


# Brightness و Contrast در پردازش تصویر

## مقدمه


 درباره‌ی **Contrast Enhancement** و روش‌هایی مانند:

* Histogram Stretching

* Histogram Equalization

* CLAHE

صحبت کردیم.



در این روش‌ها هدف اصلی ما افزایش یا اصلاح **کنتراست تصویر** بود.
اما همیشه مشکل تصویر، پایین بودن کنتراست نیست.

گاهی اوقات تصویر دارای کنتراست مناسبی است، یعنی:

* نواحی روشن را داریم.
* نواحی تاریک را داریم.
* اختلاف بین اجسام و پس‌زمینه قابل مشاهده است.

اما کل تصویر بیش از حد تاریک یا بیش از حد روشن است.

در این حالت مشکل اصلی مربوط به **Brightness** تصویر است.

# تفاوت Brightness و Intensity

برای درک بهتر موضوع، ابتدا باید تفاوت این دو مفهوم را بدانیم.

## Intensity

Intensity به مقدار روشنایی یک پیکسل خاص اشاره دارد.

برای مثال در یک تصویر خاکستری:

* مقدار 0 نشان‌دهنده‌ی سیاه کامل است.
* مقدار 255 نشان‌دهنده‌ی سفید کامل است.

هر پیکسل یک مقدار intensity دارد.

مثلاً:

```
Pixel A = 50
Pixel B = 200
```

یعنی شدت روشنایی این دو پیکسل متفاوت است.

---

## Brightness

Brightness یک مفهوم کلی‌تر است.

Brightness به روشنایی کلی تصویر اشاره می‌کند.

یعنی بررسی می‌کنیم که کل تصویر:

* تاریک است؟
* روشن است؟
* یا در شرایط نرمال قرار دارد؟

برای مثال:

فرض کنید تصویری از یک فرد در مقابل یک منبع نور گرفته شده است.

ممکن است:

* بخش‌های روشن تصویر وجود داشته باشند.
* بخش‌های تاریک نیز وجود داشته باشند.
* کنتراست تصویر مناسب باشد.

اما کل تصویر نسبت به چیزی که چشم می‌بیند، تاریک‌تر باشد.

در این حالت مشکل اصلی:

**Low Brightness**

است، نه Low Contrast.

---

# رابطه Brightness و Contrast

در تصاویر واقعی، این دو مشکل می‌توانند همزمان وجود داشته باشند.

یعنی ممکن است یک تصویر:

* Brightness پایین داشته باشد.
* Contrast پایین نیز داشته باشد.

یا:

* بیش از حد روشن باشد.
* کنتراست مناسبی نداشته باشد.

اما معمولاً یکی از این مشکلات غالب‌تر است و باید متناسب با همان مشکل روش مناسب را انتخاب کنیم.

---

# مثال: تصویر Low Brightness

فرض کنید تصویری داریم که:

* محدوده‌ی شدت روشنایی آن تقریباً کامل است.
* هم پیکسل‌های روشن داریم.
* هم پیکسل‌های تاریک داریم.

در این تصویر:

مشکل اصلی کنتراست نیست.

زیرا تفاوت بین قسمت‌های روشن و تاریک وجود دارد.

اما تصویر کمی تاریک دیده می‌شود.

در این حالت هدف ما:

**افزایش Brightness**

است.

---

# آیا روش‌های Contrast Enhancement مشکل Brightness را حل می‌کنند؟

در نگاه اول ممکن است فکر کنیم:

"اگر تصویر تاریک است، پس Equalization یا CLAHE را اجرا کنیم."

اما همیشه نتیجه مطلوب نیست.

---

## Histogram Equalization

اگر Equalization را مستقیماً روی تصویر رنگی RGB اعمال کنیم:

* کانال‌های رنگی مستقل تغییر می‌کنند.
* ترکیب رنگ‌ها به هم می‌ریزد.
* تصویر غیرواقعی می‌شود.

بنابراین Equalization مستقیم روی RGB معمولاً انتخاب مناسبی نیست.


اگر بخواهیم از Equalization استفاده کنیم، بهتر است فقط روی کانال روشنایی تصویر اعمال شود.

# استفاده از CLAHE

CLAHE یکی از روش‌هایی است که می‌تواند در چنین شرایطی کمک کند.

زیرا:

* تصویر را به صورت Local بررسی می‌کند.
* هر قسمت را متناسب با همسایه‌هایش پردازش می‌کند.
* کنتراست را کنترل‌شده افزایش می‌دهد.

اما CLAHE همچنان بیشتر برای **بهبود Contrast** طراحی شده است، نه اصلاح مستقیم Brightness.

بنابراین اگر مشکل اصلی Brightness باشد، روش‌های دیگری مناسب‌تر هستند.

# Linear Correction

یکی از روش‌های ساده برای اصلاح Brightness و Contrast استفاده از:

## Linear Intensity Transformation

است.

رابطه‌ی کلی این روش:

$$I'(x,y)=\alpha I(x,y)+\beta$$

است.

که در آن:

* $$I(x,y)$$ مقدار intensity اولیه پیکسل است.
* $$I'(x,y)$$ مقدار جدید پیکسل است.
* $$\alpha$$ ضریب تغییر Contrast است.
* $$\beta$$ مقدار تغییر Brightness است.

---

# نقش Beta در Linear Correction

اگر فقط مقدار ثابت اضافه کنیم:

$$I'(x,y)=I(x,y)+\beta$$

چه اتفاقی می‌افتد؟

تمام مقادیر تصویر به اندازه‌ی مشخصی جابه‌جا می‌شوند.

مثلاً:

قبل:

```
0  ----  50
```

بعد با:

$$\beta=60$$

تبدیل می‌شود به:

```
60 ---- 110
```

یعنی:

* تصویر روشن‌تر شده است.
* اما فاصله بین مقادیر تغییر نکرده است.

پس:

مقدار Brightness تغییر کرده، اما Contrast ثابت مانده است.

---

# نقش Alpha در Linear Correction

حال اگر:

$$I'(x,y)=\alpha I(x,y)$$

باشد.

وقتی:

$$\alpha > 1$$

مقادیر از هم فاصله بیشتری می‌گیرند.

مثلاً:

قبل:

```
0 ---- 50
```

با:

$$\alpha=2$$

تبدیل می‌شود به:

```
0 ---- 100
```

یعنی:

* فاصله intensityها بیشتر شده است.
* Contrast افزایش پیدا کرده است.

پس:

**مقدار Alpha روی Contrast اثر می‌گذارد.**

---

# مشکل Linear Correction

با وجود سادگی، این روش محدودیت‌هایی دارد.

## 1. از دست رفتن اطلاعات

محدوده شدت روشنایی تصویر محدود است.

برای تصاویر 8 بیتی:

مقدار $$0 \leq I \leq 255$$
اگر مقدار جدید از 255 بیشتر شود، مقدار آن قطع می‌شود.

مثلاً:

```
270 → 255
```

در نتیجه چند مقدار متفاوت، یک مقدار مشابه می‌شوند.

این باعث از بین رفتن جزئیات می‌شود.

<img width="1208" height="412" alt="image" src="https://github.com/user-attachments/assets/b52bd617-7101-4d8b-aedc-9a42053f6e94" />



چون وقتی الفا برابر 2 هست و ضرب میشود مثلا مقادیر 200 هم ضرب در 2 میشود و کلیپ میشود و جزییات از دست میرود.

---

## 2. افزایش بیش از حد روشنایی

اگر مقدار Alpha زیاد شود:

* قسمت‌های روشن سریعاً اشباع می‌شوند.
* جزئیات مناطق روشن از بین می‌رود.

مثلاً در تصویر یک پنجره:

قبل:

* درخت پشت پنجره قابل مشاهده است.

بعد از افزایش شدید:

* پنجره کاملاً سفید می‌شود.
* اطلاعات پشت آن حذف می‌شود.
---

با بتا مقادیر مقادبر شبفت میکنه و میتونه کنتراست ما را کاهش بده.

ما از مقدار مجاز خارج میشویم و اطلاعات را از دست میدیم.

پس حالت خطی ایراداتی داره که باید بریم سراغ روشی که بتونه بهتر عمل کنه.


---

# Gamma Correction

برای حل مشکلات Linear Correction از روش دیگری استفاده می‌کنیم:

## Gamma Correction

این روش یک تبدیل غیرخطی روی تصویر اعمال می‌کند.

فرمول کلی:

$$I' = 255(\frac{I}{255})^\gamma$$

---

# ایده Gamma Correction

ابتدا مقادیر تصویر را از بازه:

[
0-255
]

به بازه:

[
0-1
]

تبدیل می‌کنیم.

سپس مقدار Gamma را اعمال می‌کنیم.

---

# رفتار Gamma

## حالت Gamma = 1

وقتی:

$$\gamma=1$$

باشد:

[
I'=I
]

یعنی هیچ تغییری ایجاد نمی‌شود.

---

## Gamma کمتر از 1

اگر:

$$\gamma < 1$$

باشد:

مقادیر روشن‌تر می‌شوند.

مثلاً:

```
0.2 → 0.45
```

بنابراین برای تصاویر تاریک:

**مقدار Gamma کمتر از 1 باعث افزایش Brightness می‌شود.**

---

## Gamma بزرگ‌تر از 1

اگر:

$$\gamma > 1$$

باشد:

مقادیر کوچک‌تر می‌شوند.

بنابراین:

* تصویر تاریک‌تر می‌شود.
* برای تصاویر بیش از حد روشن مناسب است.

---

# Underexposed و Overexposed

در پردازش تصویر دو اصطلاح مهم داریم:

## Underexposed

تصویری که نور کمی دارد.

یعنی:

* تاریک است.
* Brightness پایین دارد.

برای اصلاح آن معمولاً:

$$\gamma < 1$$
استفاده می‌کنیم.

---

## Overexposed

تصویری که بیش از حد نور گرفته است.

یعنی:

* بسیار روشن است.
* جزئیات قسمت‌های روشن از بین رفته‌اند.

برای اصلاح آن معمولاً:

$$\gamma > 1$$

استفاده می‌کنیم.

---

# مقایسه Linear و Gamma Correction

## Linear Correction

مزایا:

* ساده
* سریع
* قابل کنترل

معایب:

* ممکن است باعث اشباع شدن پیکسل‌ها شود.
* اطلاعات قسمت‌های روشن یا تاریک از بین برود.
* تغییرات کاملاً خطی است.

---

## Gamma Correction

مزایا:

* غیرخطی است.
* محدوده 0 تا 255 را بهتر حفظ می‌کند.
* جزئیات کمتر از بین می‌روند.
* برای اصلاح Brightness مناسب‌تر است.

معایب:

* انتخاب مقدار Gamma اهمیت زیادی دارد.
* ممکن است برای بعضی تصاویر نیاز به تنظیم تجربی داشته باشد.

---

# جمع‌بندی

در بهبود تصاویر باید ابتدا تشخیص دهیم مشکل اصلی چیست:


گر مشکل **Contrast** باشد:

  * Histogram Stretching
  * Histogram Equalization
  * CLAHE

مناسب هستند.

اگر مشکل **Brightness** باشد:

  * Linear Correction
  * Gamma Correction

انتخاب مناسب‌تری هستند.

همچنین باید توجه داشت که Brightness و Contrast به یکدیگر مرتبط هستند و تغییر یکی می‌تواند روی دیگری اثر بگذارد.

در بین روش‌های اصلاح روشنایی، Gamma Correction معمولاً عملکرد بهتری دارد، زیرا بدون خارج شدن شدید از محدوده شدت روشنایی، امکان افزایش یا کاهش Brightness را فراهم می‌کند و اطلاعات تصویر را بهتر حفظ می‌کند.
