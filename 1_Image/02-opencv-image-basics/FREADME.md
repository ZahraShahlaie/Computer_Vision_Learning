
### آشنایی با ساختار تصویر در OpenCV

---

### نصب و Import

برای استفاده از OpenCV ابتدا باید آن را نصب و سپس در برنامه import کنیم:

```python
import cv2
```

---

### خواندن تصویر با imread

برای خواندن تصویر از تابع `imread` استفاده می‌کنیم:

```python
img = cv2.imread("image.jpg")
```

تصویر در OpenCV به صورت یک آرایه‌ی NumPy از نوع **ndarray** ذخیره می‌شود. این یعنی هر چیزی که روی NumPy قابل انجام است (مثل slicing، indexing و عملیات ماتریسی)، روی تصویر هم قابل اجراست.

---

### نوع داده تصویر (dtype = uint8)

زمانی که تصویر را چاپ یا بررسی می‌کنیم، معمولاً می‌بینیم:

```
dtype = uint8
```

این یعنی:

* داده‌ها از نوع عدد صحیح بدون علامت هستند
* هیچ مقدار اعشاری نداریم
* بازه مقادیر بین **0 تا 255** است

که در آن:

* 0 → سیاه
* 255 → سفید
* مقادیر بین آن → درجات مختلف شدت نور

---

### ساختار تصویر در OpenCV

تصویر در OpenCV به صورت یک ماتریس سه‌بعدی نمایش داده می‌شود:

```
(H, W, C)
```

که:

* H = ارتفاع تصویر
* W = عرض تصویر
* C = تعداد کانال‌ها (معمولاً 3 برای رنگی)

---

### تفاوت RGB و BGR در OpenCV

نکته بسیار مهم:

در بیشتر استانداردهای پردازش تصویر، ترتیب رنگ‌ها **RGB** است (Red, Green, Blue)، اما در OpenCV ترتیب به صورت **BGR** است:

```
OpenCV:   BGR
Standard:  RGB
```

یعنی:

* کانال آبی در ابتدا قرار دارد
* سپس سبز
* سپس قرمز

---

### تبدیل BGR به RGB

برای نمایش صحیح تصویر در matplotlib باید رنگ‌ها را اصلاح کنیم:

```python
img_rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
plt.imshow(img_rgb)
plt.show()
```

اگر این کار انجام نشود، رنگ‌ها اشتباه نمایش داده می‌شوند، چون:

* OpenCV: BGR می‌دهد
* matplotlib: RGB انتظار دارد

---

### نمایش تصویر خاکستری در matplotlib

اگر تصویر خاکستری باشد، باید مشخص کنیم که colormap خاکستری است:

```python
plt.imshow(img_gray, cmap='gray')
```

در غیر این صورت matplotlib ممکن است تصویر را با رنگ‌های کاذب نمایش دهد.

---

### خواندن مستقیم تصویر به صورت خاکستری

می‌توان تصویر را از ابتدا به صورت grayscale خواند:

```python
img_gray = cv2.imread("image.jpg", 0)
plt.imshow(img_gray, cmap='gray')
```

---

### تبدیل رنگ به خاکستری (RGB/BGR → Gray)

سؤال مهم: آیا تبدیل RGB به Gray و BGR به Gray یکسان است؟

پاسخ: **نه کاملاً**

چون تبدیل خاکستری فقط میانگین ساده نیست، بلکه یک تبدیل وزن‌دار است که مطابق حساسیت چشم انسان انجام می‌شود:

### فرمول استاندارد:

```
Gray = 0.299 R + 0.587 G + 0.114 B
```

نکته مهم:

* چشم انسان به رنگ سبز حساس‌تر است
* بنابراین وزن Green بیشتر است

---

### نکته مهم درباره BGR و RGB

اگر تصویر BGR باشد و مستقیم به فرمول بالا بدهیم، نتیجه دقیق نخواهد بود. به همین دلیل بهتر است:

* ابتدا BGR → RGB تبدیل شود
* سپس RGB → Gray انجام شود

---

### جمع‌بندی

در OpenCV:

* تصویر به صورت آرایه NumPy ذخیره می‌شود
* dtype معمولاً uint8 و بازه 0 تا 255 است
* ترتیب کانال‌ها BGR است نه RGB
* برای نمایش درست در matplotlib باید تبدیل رنگ انجام شود
* تبدیل خاکستری یک تبدیل وزن‌دار بر اساس ادراک چشم انسان است، نه میانگین ساده
