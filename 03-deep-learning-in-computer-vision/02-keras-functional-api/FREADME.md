#  ا Functional API در Keras

 
روش Sequential برای بسیاری از مسائل ساده کاملاً مناسب است، زیرا لایه‌ها به صورت **پشت سر هم** به شبکه اضافه می‌شوند.

اما در پروژه‌های واقعی همیشه معماری شبکه به این سادگی نیست. گاهی لازم است:

- خروجی یک لایه به چند لایه مختلف ارسال شود.
- چند مسیر مختلف در شبکه ایجاد کنیم.
- خروجی چند شاخه را با هم ترکیب کنیم.
- معماری‌هایی مانند **ResNet، U-Net، Inception** و ... را پیاده‌سازی کنیم.
    

برای چنین شرایطی از **Functional API** استفاده می‌کنیم.

---

# ا Sequential چیست؟

در کلاس Sequential، شبکه کاملاً خطی است.

هر لایه فقط خروجی لایه قبلی را دریافت می‌کند.

```
Input
   │
Conv
   │
Conv
   │
MaxPooling
   │
Conv
   │
Flatten
   │
Dense
   │
Output
```

کد آن نیز بسیار ساده است.

```python
model = Sequential()

model.add(Conv2D(...))
model.add(MaxPooling2D(...))
model.add(Flatten())
model.add(Dense(...))
```

در این حالت هیچ شاخه‌ای وجود ندارد و مسیر شبکه کاملاً مستقیم است.

---

# محدودیت Sequential

فرض کنید بخواهیم چنین معماری‌ای داشته باشیم:

```
        Conv
       /
Input
       \
        Conv

         │
      Concatenate

         │
       Output
```

یا بخواهیم خروجی یک لایه را مستقیماً به چند لایه بعدی ارسال کنیم.

در Sequential چنین چیزی امکان‌پذیر نیست.

---

# ا Functional API چیست؟

ا Functional API روشی برای ساخت شبکه‌های عصبی است که در آن **ارتباط بین لایه‌ها را خودمان مشخص می‌کنیم.**

در این روش، هر لایه مانند یک تابع عمل می‌کند.

به عبارت دیگر:

```
خروجی = لایه(ورودی)
```

مثلاً:

```python
x = Dense(128)(inputs)
```

یعنی:

- یک لایه Dense ساخته می‌شود.
- ورودی آن `inputs` است.
- خروجی داخل متغیر `x` ذخیره می‌شود.
    

در ادامه همین خروجی را به لایه بعدی می‌دهیم.

```python
x = Dense(64)(x)
```

در نهایت خروجی شبکه را تعریف می‌کنیم.

```python
outputs = Dense(10)(x)
```

---

# تفاوت اصلی Sequential و Functional API

در Sequential فقط لایه‌ها را اضافه می‌کنیم:

```python
model.add(...)
```

اما در Functional API مشخص می‌کنیم هر لایه ورودی خود را از کجا دریافت کند.

به همین دلیل ارتباط بین لایه‌ها کاملاً در اختیار برنامه‌نویس قرار دارد.

---

# مراحل ساخت یک مدل با Functional API

## مرحله اول: تعریف ورودی

ابتدا ورودی شبکه مشخص می‌شود.

```python
inputs = Input(shape=(32,32,3))
```

در این مثال:

- ارتفاع تصویر = 32
- عرض تصویر = 32
- تعداد کانال = 3 (RGB)
    

---

## مرحله دوم: ساخت لایه‌ها

هر لایه روی خروجی لایه قبلی اعمال می‌شود.

```python
x = Conv2D(...)(inputs)

x = MaxPooling2D(...)(x)

x = Conv2D(...)(x)

x = Flatten()(x)

x = Dense(...)(x)
```

دقت کنید که هر بار خروجی مرحله قبل به عنوان ورودی مرحله جدید استفاده می‌شود.

---

## مرحله سوم: تعریف خروجی

در انتها خروجی شبکه ساخته می‌شود.

```python
outputs = Dense(10, activation="softmax")(x)
```

---

## مرحله چهارم: ساخت مدل

در نهایت فقط کافی است ابتدا و انتهای شبکه را مشخص کنیم.

```python
model = Model(
    inputs=inputs,
    outputs=outputs
)
```

به همین سادگی مدل ساخته می‌شود.

---

# چرا از متغیرهایی مثل x یا block استفاده می‌کنیم؟

در Functional API معمولاً خروجی هر لایه داخل یک متغیر ذخیره می‌شود.

مثلاً:

```python
x = Conv2D(...)(inputs)

x = Conv2D(...)(x)

x = MaxPooling2D(...)(x)
```

یا

```python
block1 = Conv2D(...)(inputs)

block1 = Conv2D(...)(block1)

block1 = MaxPooling2D(...)(block1)
```

در واقع متغیر فقط **آخرین خروجی شبکه** را نگه می‌دارد.

به همین دلیل حتی اگر چند بار بنویسیم:

```python
x = ...
x = ...
x = ...
```

مشکلی ایجاد نمی‌شود، زیرا هر بار خروجی جدید جایگزین خروجی قبلی می‌شود.

به همین دلیل در بسیاری از پروژه‌های حرفه‌ای فقط از یک متغیر به نام `x` استفاده می‌شود.

---

# مثال یک شبکه CNN

فرض کنید معماری زیر را داشته باشیم:

```
Input

↓

Conv

↓

Conv

↓

MaxPooling

↓

Conv

↓

Conv

↓

MaxPooling

↓

Flatten

↓

Dense

↓

Output
```

در Functional API کافی است هر لایه را به خروجی لایه قبل متصل کنیم.

```python
inputs = Input(...)

x = Conv2D(...)(inputs)

x = Conv2D(...)(x)

x = MaxPooling2D(...)(x)

x = Conv2D(...)(x)

x = Conv2D(...)(x)

x = MaxPooling2D(...)(x)

x = Flatten()(x)

x = Dense(...)(x)

outputs = Dense(...)(x)

model = Model(inputs, outputs)
```

نتیجه دقیقاً همان شبکه Sequential خواهد بود.

---

# آیا Functional API باعث بهتر شدن مدل می‌شود؟

خیر.

اگر معماری دقیقاً یکسان باشد:

- تعداد پارامترها یکسان است.
- سرعت آموزش تقریباً یکسان است.
- دقت مدل یکسان است.    

تنها تفاوت در **روش تعریف شبکه** است.

بنابراین Functional API جایگزین Sequential نیست، بلکه روشی انعطاف‌پذیرتر برای ساخت معماری‌های پیچیده است.

---

# مثال اهمیت تعیین ورودی هر لایه

فرض کنید سه بلاک داریم.

```
Input

↓

Block1

↓

Block2

↓

Block3

↓

Output
```

تعریف صحیح:

```python
block1 = ...

block2 = ...(block1)

block3 = ...(block2)
```

در این حالت اطلاعات از هر سه بلاک عبور می‌کند.

اما اگر اشتباه بنویسیم:

```python
block3 = ...(block1)
```

در این حالت خروجی Block2 کاملاً نادیده گرفته می‌شود.

مسیر واقعی شبکه به شکل زیر خواهد بود:

```
Input

↓

Block1

↘

 Block3

↓

Output
```

یعنی Block2 عملاً از شبکه حذف شده است.

به همین دلیل هنگام استفاده از Functional API باید دقت کنیم که ورودی هر لایه دقیقاً از کدام خروجی دریافت می‌شود.

---

# ارتباط Functional API با ResNet

یکی از مهم‌ترین کاربردهای Functional API پیاده‌سازی معماری‌هایی مانند **ResNet** است.

در شبکه‌های بسیار عمیق، گرادیان هنگام عبور از تعداد زیادی لایه ممکن است ضعیف شود و یادگیری برخی لایه‌ها کاهش یابد.

ا ResNet با استفاده از **Skip Connection** این مشکل را تا حد زیادی برطرف می‌کند.

```
Input
   │
Conv
   │
Conv
   │
 +───────────────+
 │               │
 +────── Add ◄───+
        │
      Output
```

در این معماری خروجی یک لایه مستقیماً به لایه‌های جلوتر ارسال می‌شود.

چنین ساختاری با Sequential قابل پیاده‌سازی نیست و به همین دلیل از Functional API استفاده می‌شود.

---

# آموزش مدل

پس از ساخت مدل، ادامه مراحل دقیقاً مشابه Sequential است.

```python
model.compile(...)

model.fit(...)

model.evaluate(...)
```

هیچ تفاوتی در آموزش مدل وجود ندارد.

---

# نکته درباره Dropout

در لایه‌های Convolution معمولاً مقدار Dropout زیاد انتخاب نمی‌شود، زیرا وظیفه این لایه‌ها استخراج ویژگی است.

معمولاً مقادیر زیر مناسب هستند:

- بین **0.1 تا 0.2** برای لایه‌های Convolution
- حدود **0.5** برای لایه‌های Dense
    

البته مقدار مناسب به نوع مسئله و دیتاست بستگی دارد.

---

# جمع‌بندی

ا Functional API روشی قدرتمند برای طراحی شبکه‌های عصبی در Keras است که به برنامه‌نویس اجازه می‌دهد ارتباط بین لایه‌ها را به صورت دلخواه تعریف کند. برخلاف Sequential که تنها از یک مسیر خطی پشتیبانی می‌کند، در Functional API می‌توان چندین مسیر مختلف ایجاد کرد، خروجی یک لایه را به چند بخش ارسال نمود یا چند شاخه را با یکدیگر ادغام کرد.

اگر معماری شبکه ساده و خطی باشد، خروجی Functional API و Sequential کاملاً یکسان خواهد بود. اما برای شبکه‌های پیشرفته‌ای مانند **ResNet، U-Net، Inception، Siamese Network** و بسیاری از معماری‌های مدرن، استفاده از Functional API ضروری است.

به طور خلاصه، می‌توان گفت:

- ا **Sequential** برای شبکه‌های ساده و خطی مناسب است.
    
- ا **Functional API** برای شبکه‌های پیچیده، چندشاخه و دارای ارتباطات غیرخطی طراحی شده و انعطاف بسیار بیشتری در اختیار برنامه‌نویس قرار می‌دهد.
