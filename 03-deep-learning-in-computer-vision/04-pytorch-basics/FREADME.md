# آشنایی با PyTorch 



 ‏PyTorch یک فریم‌ورک متن‌باز برای یادگیری عمیق است که توسط Meta (Facebook AI) توسعه داده شده است.

این کتابخانه امکانات کاملی برای:

- ساخت شبکه‌های عصبی
- آموزش مدل‌ها
- پردازش تصاویر
- پردازش متن
- پردازش صوت
- محاسبات GPU


در اختیار برنامه‌نویسان قرار می‌دهد.

امروزه بسیاری از مقالات علمی و پروژه‌های تحقیقاتی از PyTorch استفاده می‌کنند.

---

#  ‏TensorFlow یا PyTorch؟

هر دو فریم‌ورک بسیار قدرتمند هستند و تقریباً تمام مدل‌های یادگیری عمیق را می‌توان با هر دو پیاده‌سازی کرد.

به طور کلی:

###  ‏TensorFlow

- مناسب پروژه‌های صنعتی
- ابزارهای آماده فراوان
- استقرار (Deployment) ساده‌تر
- ادغام مناسب با Keras

###  ‏PyTorch

- کدنویسی ساده‌تر
- انعطاف‌پذیری بیشتر
- محبوب در تحقیقات علمی
- استفاده گسترده در مقالات جدید

در عمل یادگیری هر دو فریم‌ورک یک مزیت مهم برای هر متخصص هوش مصنوعی محسوب می‌شود.

---

# نسخه Stable یا Nightly؟

هنگام نصب دو گزینه وجود دارد.

###  ‏Stable

نسخه پایدار

- تست شده
- بدون باگ‌های مهم
- مناسب پروژه‌های واقعی
    

###  ‏Nightly

نسخه آزمایشی

- شامل قابلیت‌های جدید
- ممکن است ناپایدار باشد
- مناسب توسعه‌دهندگان PyTorch    

برای اکثر پروژه‌ها همیشه نسخه **Stable** پیشنهاد می‌شود.

# ساختار داده در PyTorch

یکی از تفاوت‌های مهم PyTorch و TensorFlow نحوه ذخیره تصاویر است.

---

##  ‏TensorFlow

ترتیب ابعاد تصویر:

```text
Height × Width × Channel
```

یا

```text
H × W × C
```

مثال:

```text
224 × 224 × 3
```

---

##  ‏PyTorch

ترتیب ابعاد:

```text
Channel × Height × Width
```

یا

```text
C × H × W
```

مثال:

```text
3 × 224 × 224
```

این تفاوت را همیشه باید در ذهن داشته باشیم.

#  ‏DataLoader چیست؟

تا اینجا فقط دیتاست را خواندیم.

اما شبکه عصبی نمی‌تواند کل دیتاست را یکجا پردازش کند.

به همین دلیل داده‌ها به دسته‌های کوچک‌تر تقسیم می‌شوند.

این دسته‌ها را **Batch** می‌گوییم.

مسئول این کار کلاس

```python
DataLoader
```

است.

---

# ساخت DataLoader

```python
train_loader = torch.utils.data.DataLoader(...)
```

این کلاس دیتاست را دریافت می‌کند و آن را به Batchهای کوچک تقسیم می‌کند.

- ‏Batch Size: یکی از مهم‌ترین پارامترهای DataLoader است.

مثلاً:

```python
batch_size = 64
```

یعنی در هر مرحله آموزش، ۶۴ تصویر وارد شبکه می‌شود.

-  ‏Shuffle: در داده‌های آموزشی معمولاً از

```python
shuffle=True
```

استفاده می‌کنیم.

این کار باعث می‌شود ترتیب تصاویر در هر Epoch به صورت تصادفی تغییر کند.

مزایای آن:

- جلوگیری از یادگیری ترتیب داده‌ها
- افزایش قدرت تعمیم مدل    
- کاهش احتمال بیش‌برازش (Overfitting)
    

---

در داده‌های تست معمولاً:

```python
shuffle=False
```

است.

زیرا نیازی به تغییر ترتیب داده‌ها وجود ندارد.

---

```python
model.fit(...)
```

اما در PyTorch Batchها توسط DataLoader ساخته می‌شوند و شبکه این Batchها را دریافت می‌کند.

بنابراین مدیریت داده در PyTorch انعطاف بیشتری دارد.

# ساخت شبکه عصبی در PyTorch

در PyTorch تمامی لایه‌های شبکه عصبی داخل ماژول `torch.nn` قرار دارند.

ابتدا آن را Import می‌کنیم.

```python
from torch import nn
```

کلمه **nn** مخفف **Neural Network** است.

---

# ساخت مدل Sequential

ساده‌ترین روش ساخت شبکه در PyTorch استفاده از `Sequential` است.

```python
model = nn.Sequential(
    ...
)
```

این ساختار دقیقاً مشابه Keras است؛ یعنی خروجی هر لایه، ورودی لایه بعدی خواهد بود.

---

# اضافه کردن اولین لایه Convolution

برای تصاویر از لایه‌های کانولوشنی استفاده می‌کنیم.

```python
nn.Conv2d(...)
```

پارامترهای اصلی این لایه عبارت‌اند از:

```python
nn.Conv2d(
    in_channels,
    out_channels,
    kernel_size,
    padding
)
```

---

## 1) in_channels

تعداد کانال‌های ورودی تصویر را مشخص می‌کند.

برای دیتاست MNIST:

```
28 × 28 × 1
```

تصویر خاکستری است، بنابراین:

```python
in_channels = 1
```

---

## 2) out_channels

تعداد Feature Mapهای خروجی است.

در واقع:

> تعداد Kernelهایی که در این لایه داریم.

مثلاً اگر بنویسیم:

```python
out_channels = 16
```

یعنی

```
16 Kernel
```

خواهیم داشت.

در نتیجه خروجی شبکه به صورت زیر خواهد بود:

```
28 × 28 × 16
```

(در صورتی که Padding مناسب انتخاب شده باشد.)

---

## 3) kernel_size

اندازه Kernel را مشخص می‌کند.

مثلاً

```python
kernel_size = 3
```

یعنی

```
3×3
```

که رایج‌ترین انتخاب در CNNها است.

---

## 4) padding

Padding مشخص می‌کند که آیا اطراف تصویر صفر اضافه شود یا خیر.

مثلاً:

```python
padding = 1
```

باعث می‌شود اندازه تصویر حفظ شود.

---

# چرا Activation داخل Conv ننوشته‌ایم؟

در Keras معمولاً می‌نوشتیم:

```python
Conv2D(..., activation="relu")
```

اما در PyTorch معمولاً Activation را به صورت یک لایه جداگانه می‌نویسیم.

مثلاً

```python

nn.ReLU()
```

علت این است که در PyTorch هر مرحله به صورت مستقل تعریف می‌شود.

ابتدا:

```
Convolution
```

سپس

```
Activation
```

---

# اضافه کردن MaxPooling

مدل تصاویر را دریافت می‌کند و خروجی را پیش‌بینی می‌کند.

---

# محاسبه Loss

```python
loss = criterion(outputs,labels)
```

این تابع اختلاف بین خروجی مدل و Label واقعی را محاسبه می‌کند.

---

# صفر کردن Gradient

قبل از هر Batch باید بنویسیم:

```python
optimizer.zero_grad()
```

چون در PyTorch گرادیان‌ها به صورت تجمعی ذخیره می‌شوند و اگر پاک نشوند، آموزش مدل دچار خطا خواهد شد.

---

# Backpropagation

حال باید گرادیان‌ها محاسبه شوند.

```python
loss.backward()
```

این همان الگوریتم Backpropagation است.

```python

```

وزن‌های شبکه را به‌روزرسانی می‌کند.

```python
running_loss = 0
```

سپس:

```python

```

در پایان Epoch معمولاً میانگین گرفته می‌شود.

# تفاوت مهم PyTorch و TensorFlow

|TensorFlow / Keras|PyTorch|
|---|---|
|`model.fit()` آموزش را انجام می‌دهد|حلقه آموزش باید دستی نوشته شود.|
|Softmax معمولاً داخل مدل تعریف می‌شود.|هنگام استفاده از `CrossEntropyLoss` نباید Softmax را داخل مدل قرار داد.|
|`model.summary()` وجود دارد.|از `torchsummary` برای مشاهده ساختار مدل استفاده می‌شود.|
|بسیاری از مراحل به صورت خودکار انجام می‌شوند.|تمام مراحل مانند Forward، Loss، Backward و Update به صورت دستی کنترل می‌شوند.|

# ارزیابی (Evaluation) مدل

در یادگیری ماشین تنها بالا بودن دقت روی داده‌های آموزشی کافی نیست.

هدف اصلی این است که مدل بتواند روی داده‌هایی که قبلاً ندیده نیز عملکرد خوبی داشته باشد.

به همین دلیل معمولاً داده‌ها را به دو بخش تقسیم می‌کنیم:

- **Train Dataset** → برای آموزش مدل
    
- **Test Dataset** → برای ارزیابی مدل

# تفاوت train و eval

این دستور مدل را وارد **حالت آموزش** می‌کند.

اما هنگام ارزیابی باید مدل را وارد حالت **Evaluation** کنیم.

```python
model.eval()
```

این کار باعث می‌شود لایه‌هایی مانند:

- ا Dropout

بنابراین همیشه قبل از ارزیابی باید بنویسیم:


# ارزیابی روی داده‌های تست

# چرا Softmax داخل مدل وجود ندارد؟

در TensorFlow معمولاً آخرین لایه به صورت زیر است:

```python
Dense(10, activation="softmax")
```

اما در PyTorch معمولاً چنین چیزی وجود ندارد.

به جای آن می‌نویسیم:

```python
nn.Linear(...)
```

و از Loss زیر استفاده می‌کنیم:

```python
nn.CrossEntropyLoss()
```

دلیل آن این است که:

> ا **CrossEntropyLoss خودش عملیات Softmax را به صورت داخلی انجام می‌دهد.**

بنابراین اگر دوباره Softmax داخل مدل قرار دهیم، محاسبات اشتباه خواهد شد.

---

# خروجی مدل چیست؟

از آنجایی که داخل مدل Softmax نداریم، خروجی مدل احتمال (Probability) نیست.

بلکه خروجی از نوع **Logits** است.

مثلاً ممکن است خروجی مدل چنین چیزی باشد:

```
[-2.1, 5.8, -1.4, 0.9, ...]
```

این اعداد احتمال نیستند.

---

# تبدیل خروجی به Probability

اگر بخواهیم احتمال هر کلاس را ببینیم باید خودمان Softmax را اعمال کنیم.

```python
probabilities = nn.softmax(dim=1)(outputs)
```

اکنون خروجی چیزی شبیه زیر خواهد بود:

```
[0.001,
 0.003,
 0.982,
 0.010,
 ...]
```

جمع تمام این اعداد برابر ۱ است.

---

# آیا در Production هم باید Softmax را اعمال کنیم؟

بله.

در زمان آموزش نیازی به Softmax داخل مدل نیست.

اما اگر بخواهیم:

- احتمال هر کلاس را نمایش دهیم
- ا Confidence مدل را بدانیم
- خروجی را به کاربر نشان دهیم
    

می‌توانیم بعد از خروجی مدل Softmax را اعمال کنیم.

زیرا Softmax هیچ پارامتر آموزشی ندارد.

# چرا به کلاس (Class) نیاز داریم؟

در این شرایط از برنامه‌نویسی شی‌گرا (Object-Oriented Programming) استفاده می‌کنیم.

---

# ساخت مدل با کلاس

ابتدا از nn.Module ارث‌بری می‌کنیم.

```python
class CNN(nn.Module):
```

به همین دلیل تمام قابلیت‌های PyTorch به کلاس ما اضافه می‌شود.

---

# تابع **init**

داخل این تابع تمام لایه‌ها تعریف می‌شوند.

مثلاً:

```python
self.conv1 = nn.Conv2d(...)
```

یا

```python
self.pool1 = nn.MaxPool2d(...)
```

یا

```python
self.fc1 = nn.Linear(...)
```

در این قسمت فقط ساختار مدل تعریف می‌شود.

---

# تابع forward

در این تابع مشخص می‌کنیم داده چگونه داخل شبکه حرکت کند.

مثلاً:

```python
x = self.conv1(x)

x = self.relu1(x)

x = self.pool1(x)

x = self.conv2(x)

x = self.flatten(x)

x = self.fc1(x)

return x
```

در واقع مسیر عبور داده از شبکه در این قسمت مشخص می‌شود.

---

# مزیت استفاده از کلاس

ساخت مدل با کلاس مزایای زیادی دارد:

- امکان ساخت شبکه‌های بسیار پیچیده
- پیاده‌سازی Skip Connection
- ساخت Residual Block
- استفاده مجدد از Blockها
- خوانایی بیشتر کد
- توسعه آسان‌تر پروژه
    

به همین دلیل تقریباً تمام مدل‌های حرفه‌ای PyTorch با استفاده از **Class** نوشته می‌شوند.

مفاهیم پایه‌ی PyTorch و ساختار کلی آن آشنا شدیم. یاد گرفتیم چگونه PyTorch را نصب کنیم، کتابخانه‌های اصلی آن را وارد پروژه کنیم و با استفاده از `torchvision` دیتاست آماده‌ی MNIST را بارگذاری کنیم.

با فرآیند کامل ساخت و آموزش یک شبکه عصبی کانولوشنی در PyTorch آشنا شدیم. ابتدا با استفاده از `nn.Sequential` معماری شبکه شامل لایه‌های **Conv2D، ReLU، MaxPooling، Flatten و Linear** را ساختیم و یاد گرفتیم که در PyTorch تابع فعال‌سازی به صورت لایه‌ای جداگانه تعریف می‌شود. سپس دیدیم که برای مسائل دسته‌بندی چندکلاسه، استفاده از `CrossEntropyLoss` کافی است و نیازی به قرار دادن `Softmax` در انتهای مدل نیست، زیرا این تابع آن را به صورت داخلی اعمال می‌کند.

در ادامه، نحوه تعریف **Optimizer** و نقش آن در به‌روزرسانی وزن‌ها را بررسی کردیم و مهم‌ترین تفاوت PyTorch با Keras را مشاهده کردیم؛ اینکه در PyTorch خبری از `model.fit()` نیست و باید کل فرآیند آموزش شامل **Forward Pass، محاسبه Loss، Backpropagation، به‌روزرسانی وزن‌ها و محاسبه معیارهایی مانند Accuracy** را به صورت دستی پیاده‌سازی کنیم. این رویکرد اگرچه کدنویسی بیشتری نیاز دارد، اما کنترل بسیار بیشتری روی فرآیند آموزش در اختیار ما قرار می‌دهد و یکی از دلایل محبوبیت PyTorch در پروژه‌های پژوهشی و پیشرفته یادگیری عمیق است.
