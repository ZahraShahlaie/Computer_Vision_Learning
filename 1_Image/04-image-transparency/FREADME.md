

# 🖼️ فرمت تصاویر، کانال‌ها و شفافیت (JPEG vs PNG)

## 📌 مقدمه

در پردازش تصویر، شناخت ساختار فایل‌های تصویری مثل **JPEG و PNG** و نحوه‌ی ذخیره‌سازی کانال‌های رنگی و شفافیت (Transparency) بسیار مهم است.

بسیاری از خطاها در OpenCV دقیقاً از عدم درک همین تفاوت‌ها بین کانال‌ها (3 کاناله و 4 کاناله) ایجاد می‌شود.



---



# 📷 JPEG ?



## ✔️ ویژگی‌ها



* فشرده‌سازی Lossy (با افت کیفیت)
* مناسب برای عکس‌های واقعی
* سبک و کم‌حجم



## 🎨 ساختار کانال‌ها



JPEG معمولاً فقط شامل:



* Red
* Green
* Blue



📌 یعنی:



> هر پیکسل = 3 مقدار (RGB)



❌ JPEG
از **شفافیت (Alpha Channel)** پشتیبانی نمی‌کند.



---



# 🖼️ PNG?



## ✔️ ویژگی‌ها



* فشرده‌سازی Lossless (بدون افت کیفیت)
* مناسب برای تصاویر گرافیکی، UI، ماسک‌ها
* پشتیبانی از شفافیت (Transparency)



## 🎨 حالت‌های رایج PNG



### 1️⃣ RGB (3 کاناله)



* Red
* Green
* Blue



### 2️⃣ RGBA (4 کاناله)



* Red
* Green
* Blue
* Alpha (شفافیت)



---



# 🌈 Alpha Channel?



 یک کانال اضافه است که میزان شفافیت هر پیکسل را مشخص می‌کند.



## 📊 محدوده مقدارها:



| مقدار | معنی                    |
| ----- | ----------------------- |
| 0     | کاملاً شفاف (Invisible) |
| 255   | کاملاً غیرشفاف (Opaque) |





📌 نکته مهم:



> Alpha = شدت دیده شدن پیکسل



---



# 🧠 مفهوم Transparency در عمل



شفافیت یعنی هر پیکسل چقدر از پس‌زمینه تأثیر بگیرد.



* Alpha = 0 → پیکسل دیده نمی‌شود
* Alpha = 255 → پیکسل کامل دیده می‌شود
* بین این دو → نیمه شفاف



📌 در پردازش تصویر:



> Alpha نقش ماسک (Mask) را دارد



---



# ⚠️ مشکل مهم OpenCV: 4 کاناله بودن تصاویر



خواندن تصاویر 4 کاناله PNG با OpenCV:



```python

cv2.imread(path, cv2.IMREAD_UNCHANGED)

```



---



## ❗ مشکل چیست؟



بسیاری از توابع OpenCV و حتی بعضی عملیات‌های NumPy:



* فقط با 3 کانال (BGR) کار می‌کنند
* و با 4 کاناله دچار خطا یا رفتار غیرمنتظره می‌شوند



---



# 🔄 چرا 4 کاناله داریم؟



فرمت PNG می‌تواند شامل Alpha باشد:



* 3 کانال → تصویر معمولی
* 4 کانال → تصویر + شفافیت



📌 پس:



> کانال چهارم = Alpha Channel



---

# 🛠️ راه‌حل‌های مشکل 4 کاناله

## ✅ راه‌حل 1: حذف Alpha Channel

اگر شفافیت مهم نیست:

* فقط 3 کانال RGB/BGR نگه داشته می‌شود
* کانال Alpha حذف می‌شود

📌 نتیجه:

> تصویر به حالت معمولی رنگی تبدیل می‌شود

---

## ✅ راه‌حل 2: استفاده از Alpha به عنوان ماسک

اگر شفافیت مهم است:

* Alpha را جدا می‌کنیم
* از آن به عنوان Mask استفاده می‌کنیم

📌 کاربرد:

* segmentation
* blending
* overlay تصاویر

---

## ✅ راه‌حل 3: ترکیب (Blending) با پس‌زمینه

وقتی تصویر شفاف داریم:

* پیکسل‌ها با پس‌زمینه ترکیب می‌شوند
* بر اساس مقدار Alpha

📌 مفهوم:

> هرچه Alpha کمتر → تأثیر بیشتر پس‌زمینه

---


# 🧩 تفاوت مهم RGB و BGR در OpenCV

OpenCV به جای RGB از:

* BGR استفاده می‌کند

📌 یعنی:

* Blue
* Green
* Red

❗ اگر این نکته رعایت نشود:

> رنگ‌ها اشتباه نمایش داده می‌شوند

---

# 📌 جمع‌بندی

* JPEG → فقط RGB (3 کاناله)، بدون شفافیت
* PNG → می‌تواند RGBA (4 کاناله) باشد
* Alpha → میزان شفافیت هر پیکسل
* OpenCV → ممکن است BGRA بخواند
* مشکل اصلی → ناسازگاری 3 کاناله و 4 کاناله
* راه‌حل → حذف Alpha یا استفاده از آن به عنوان ماسک
